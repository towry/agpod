# Agpod Flow

## When to Use

Load this skill when the user mentions:
- `agpod flow`, `flow graph`, `task graph`, `rebuild graph`
- Task navigation: focus, fork, parent, tree, recent tasks
- Flow sessions: create/list/close
- Document mounting or frontmatter initialization for flow
- 任务状态、当前任务、task status、what's the current task — 使用 `agpod flow -s <id> status` 或 `agpod flow recent`

## Core Rules

- Stateful commands require `-s <session-id>`.
- Stateless commands (`rebuild`, `recent`, `tree`) do not require `-s`.
- `doc_id` is generated by `doc init`/`doc add`, not manually.
- Flow docs must have valid YAML frontmatter with `doc_id`.
- **Mandatory**: Do not create or mount agpod flow docs manually with `cat`, heredoc, shell redirection (`>`, `>>`), or hand-written frontmatter. You must use `agpod flow doc init|add|remove` CLI commands.
- `doc --path` must be a relative path from repo root; absolute paths are not allowed.
- **Mandatory**: Do not perform manual checks (for example, manually inspecting doc paths/frontmatter/graph files). Use `agpod flow` CLI commands for all verification and validation.
- Fast-path default: do not run `agpod flow init` on every use. Only run `agpod flow init` when commands fail and indicate missing flow config/bootstrap state (or when initialization is explicitly requested).

## Command Reference

### Stateless

```bash
agpod flow rebuild                    # Rebuild graph.json from documents
agpod flow recent -n 10 --days 14     # List recent tasks
agpod flow tree                       # Print full task tree
agpod flow tree --root T-001          # Print subtree
```

### Session Lifecycle

```bash
agpod flow session new                          # Create session
agpod flow session list                         # List sessions
agpod flow -s <session-id> session close        # Close session
```

### Stateful (require -s)

```bash
agpod flow -s <id> status                                       # Show active task
agpod flow -s <id> focus --task T-001.2                         # Focus on task
agpod flow -s <id> fork --checkpoint "doc updated"              # Fork sub-task (auto allocates task id)
agpod flow -s <id> fork --from T-001.2 --checkpoint "doc updated" --no-switch
agpod flow -s <id> parent                                       # Navigate to parent
```

### Document Operations

```bash
# Initialize frontmatter + write content (--doc-type / --content required, --task optional)
agpod flow doc init --path docs/design.md --doc-type design --task T-001.2 --content "# Design\n\nDetails"

# Mount doc to task + write content (requires -s, --task defaults to active task)
agpod flow -s <id> doc add --path docs/impl.md --content "# Impl\n\nDetails"
agpod flow -s <id> doc add --path docs/note.md --task T-001.3 --doc-type note --content "# Note\n\nDetails"

# Unmount doc from flow graph (strip frontmatter, keep file body)
agpod flow doc remove --path docs/note.md
```

## Workflow

1. `agpod flow rebuild` — rebuild graph after docs change
2. `agpod flow recent -n 5` — identify target tasks
3. `agpod flow session new` — create session
4. `agpod flow -s <id> focus --task <task>` — focus
5. `agpod flow -s <id> fork --checkpoint "<summary>"` — fork if needed (CLI allocates child task id)
6. `agpod flow doc init ...` — attach docs

## Troubleshooting

| Error | Fix |
|-------|-----|
| `requires -s <session-id>` | `agpod flow session new` |
| `No active task in session` | `agpod flow -s <id> focus --task <task-id>` |
| `graph.json not found` | `agpod flow rebuild` |
| `Frontmatter missing` / `doc_id missing` | `agpod flow doc init --path <file> --doc-type <type>` |
