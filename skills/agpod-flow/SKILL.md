# Agpod Flow

## When to Use

Use this skill when the user asks to:
- rebuild flow graph from docs
- inspect recent tasks or task tree
- create/list/close flow sessions
- focus/fork/parent task navigation
- initialize or mount documents into flow task graph
- debug flow failures (`frontmatter missing`, `doc_id missing`, `session not found`)

## Core Rules

- Use explicit session for stateful commands: always pass `-s <session-id>`.
- Do not rely on implicit active task.
- Stateless commands (`rebuild`, `recent`, `tree`) do not require `-s`.
- `doc_id` is generated by tool commands (`doc init`/`doc add`), not manually.
- Flow docs must have valid YAML frontmatter; missing frontmatter/doc_id must fail fast.
- `repo-id` is derived from git remote URL, not local path.

## Command Map

### Stateless

```bash
agpod flow rebuild
agpod flow recent -n 10 --days 14
agpod flow tree
agpod flow tree --root T-001
```

### Session Lifecycle

```bash
agpod flow session new
agpod flow session list
agpod flow -s <session-id> session close
```

### Stateful

```bash
agpod flow -s <session-id> status
agpod flow -s <session-id> focus --task T-001.2
agpod flow -s <session-id> fork --to T-001.2.1
agpod flow -s <session-id> fork --to T-001.2.1 --from T-001.2 --no-switch
agpod flow -s <session-id> parent
```

### Document Operations

```bash
agpod flow -s <session-id> doc init --path docs/design.md --task T-001.2 --type design
agpod flow -s <session-id> doc add --path docs/impl.md
agpod flow -s <session-id> doc add --path docs/note.md --task T-001.3 --type note
```

## Recommended Workflow

1. Rebuild graph first when repository docs changed.
2. Use `recent` to identify candidate tasks quickly.
3. Create a session and focus on target task.
4. Fork sub-task only when needed, then continue with focused implementation.
5. Use `doc init` for new docs and `doc add` to mount/update metadata.

Example:

```bash
agpod flow rebuild
agpod flow recent -n 5
agpod flow session new
agpod flow -s S-xxxxxx focus --task T-001.2
agpod flow -s S-xxxxxx fork --to T-001.2.1
agpod flow -s S-xxxxxx doc init --path docs/fix.md --task T-001.2.1 --type impl
```

## Fail-Fast Troubleshooting

- `This command requires -s <session-id>`:
  - Create session: `agpod flow session new`
  - Retry with `-s`.
- `No active task in session ...`:
  - `agpod flow -s <id> focus --task <task-id>`
- `graph.json not found ...`:
  - `agpod flow rebuild`
- `Frontmatter missing` or `doc_id missing`:
  - `agpod flow -s <id> doc init --path <file> --task <task-id> --type <doc-type>`
